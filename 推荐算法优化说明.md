# 推荐算法优化说明

## 问题描述

之前的推荐算法太简单，只是根据单个关键词匹配，导致推荐不准确。

## 优化方案

### 新的推荐逻辑

系统现在根据用户的**健康状态**，使用**多关键词匹配**来精准推荐内容。

---

## 推荐策略

### 1️⃣ 高风险用户（建议沟通）

**用户特征**: 测试结果显示需要专业关注

**推荐科普关键词**（匹配任意一个即可）:
```
抑郁、情绪、压力、焦虑、疏导、放松、调节、心理
```

**推荐活动关键词**（匹配任意一个即可）:
```
冥想、放松、减压、瑜伽、音乐、艺术、团辅
```

**推荐内容示例**:
- 科普: "情绪管理指南"、"如何缓解焦虑"、"压力调节技巧"
- 活动: "冥想放松工作坊"、"音乐减压活动"、"瑜伽体验课"

**额外建议**: 推荐AI心理咨询

---

**推荐科普关键词**:
```
健康、预防、调节、情绪、压力、心理、成长
```

**推荐活动关键词**:

**推荐内容示例**:
- 科普: "心理健康预防"、"情绪调节方法"、"压力管理"
- 活动: "心理健康讲座"、"户外拓展活动"、"团体辅导"

---

### 3️⃣ 低风险用户（健康）

**用户特征**: 测试结果显示心理状态良好

**推荐科普关键词**:
```
成长、提升、积极、阳光、心理、健康
```

**推荐活动关键词**:

**推荐内容示例**:
- 科普: "积极心理学"、"个人成长指南"、"阳光心态培养"
- 活动: "志愿者活动"、"户外运动会"、"社团交流活动"

## 技术实现

### 关键词匹配逻辑

```java
// 高风险用户推荐
if (healthStatus == NEED_TALK) {
    // 匹配任意一个关键词即可
    wrapper.and(w -> {
        w.like(Knowledge::getTitle, "抑郁")
         .or().like(Knowledge::getSummary, "抑郁")
         .or().like(Knowledge::getTitle, "情绪")
         .or().like(Knowledge::getSummary, "情绪")
         // ... 其他关键词
    });
}
```

### 降级策略

如果根据关键词没有找到足够的内容，系统会：
1. 记录警告日志
2. 返回热门内容作为备选
3. 确保用户始终能看到推荐内容

## 推荐效果对比

### 优化前
- ❌ 只匹配单个关键词（如"减压"）
- ❌ 推荐内容单一
- ❌ 没有根据健康状态区分
- ❌ 活动推荐不准确

### 优化后
- ✅ 多关键词匹配（8个关键词）
- ✅ 推荐内容丰富多样
- ✅ 根据健康状态精准推荐
- ✅ 活动推荐更贴合用户需求
- ✅ 添加日志记录，便于调试

## 日志监控

系统会记录推荐日志，便于管理员监控推荐效果：

```
INFO  - 为高风险用户 [userId=123] 生成推荐
INFO  - 根据关键词 [抑郁, 情绪, 压力, ...] 找到 5 条知识
INFO  - 根据关键词 [冥想, 放松, 减压, ...] 找到 3 条活动
```

如果未找到匹配内容：
```
WARN  - 根据关键词 [冥想, 放松, 减压, ...] 未找到活动，返回热门活动
```

## 后续优化方向

1. **用户画像**: 根据用户历史行为（浏览、收藏、评论）优化推荐
2. **协同过滤**: 基于相似用户的喜好进行推荐
3. **内容标签**: 为科普和活动添加更详细的标签
4. **A/B测试**: 测试不同推荐策略的效果
5. **实时反馈**: 根据用户点击行为动态调整推荐
6. **时间因素**: 考虑活动开始时间，优先推荐近期活动

### 活动
