# 高危预警用户判定标准说明

## 判定标准

系统通过**多维度数据分析**来判定高危预警用户，主要包括以下几个方面：

---

### 1️⃣ AI咨询情绪分析（实时）

**判定逻辑**：在用户与AI助手聊天时，实时分析用户消息中的关键词

#### 高风险关键词（匹配1个即预警）
```
自杀、不想活、去死、结束生命、跳楼、割腕、
绝望、崩溃、活不下去、没意思、活着没意义
```

**风险等级**: HIGH（高）

**触发条件**: 消息中包含任意1个高风险关键词

**示例**:
- 用户说："我感觉活着没意思"
- 系统检测到"活着没意义"关键词
- 立即创建高风险预警

---

#### 中风险关键词（匹配2个及以上才预警）
```
抑郁、焦虑、痛苦、无助、孤独、没人理解、
压力大、失眠、哭泣、难过、伤心、害怕、
紧张、恐惧、担心、烦躁、愤怒、生气
```

**风险等级**: MEDIUM（中）

**触发条件**: 消息中包含2个及以上中风险关键词

**示例**:
- 用户说："我最近很焦虑，而且经常失眠"
- 系统检测到"焦虑"和"失眠"两个关键词
- 创建中风险预警

---

### 2️⃣ 测试结果分析（测试提交时）

**判定逻辑**：用户完成心理测试后，根据测试结果判定

#### 高风险
- **条件**: 测试结果 = "建议沟通"（分数 ≥ 良好阈值）
- **风险等级**: HIGH
- **说明**: 用户心理状态需要专业关注

#### 中风险
- **条件**: 测试结果 = "良好"（分数在健康阈值和良好阈值之间）
- **风险等级**: MEDIUM
- **说明**: 用户心理状态需要适当关注

---

### 3️⃣ 健康状态持续恶化（定期分析）

**判定逻辑**：分析用户历史测试记录，判断健康状态趋势

#### 高风险
- **条件**: 连续3次测试结果都是"建议沟通"
- **风险等级**: HIGH
- **说明**: 用户心理状态持续恶化，需要紧急干预

#### 中风险
- **条件**: 最近2次测试结果都是"建议沟通"
- **风险等级**: MEDIUM
- **说明**: 用户心理状态有恶化趋势

---

### 4️⃣ 综合风险评估

系统会根据以上多个维度综合评估用户风险等级：

| 维度 | 高风险 | 中风险 |
|-----|--------|--------|
| AI情绪分析 | 匹配高风险关键词 | 匹配2个+中风险关键词 |
| 测试结果 | "建议沟通" | "良好" |
| 健康趋势 | 连续3次恶化 | 连续2次恶化 |

**最终风险等级判定**:
- 任意维度为HIGH → 最终风险等级为HIGH
- 所有维度都为MEDIUM → 最终风险等级为MEDIUM
- 所有维度都为LOW → 无预警

---

## 预警处理流程

### 1. 预警创建
- 系统自动检测并创建预警记录
- 记录用户ID、风险等级、触发关键词、创建时间

### 2. 预警展示
- 管理员在"数据统计"页面查看预警列表
- 显示用户详细信息和风险等级

### 3. 预警处理
- 管理员点击"处理"按钮
- 填写处理备注（如：已联系用户、已安排心理咨询等）
- 标记为已处理

### 4. 后续跟进
- 定期回访用户
- 记录用户心理状态变化
- 更新预警状态

## 技术实现

### 后端实现

#### 情绪分析服务
```java
@Service
public class EmotionAnalysisServiceImpl implements EmotionAnalysisService {
    
    @Override
    public void analyzeAndWarn(Long userId, String message) {
        // 1. 检查高风险关键词
        if (matchedHighRisk) {
            warningService.createWarning(userId, "HIGH", keywords);
        }
        
        // 2. 检查中风险关键词
        if (matchedMediumRisk >= 2) {
            warningService.createWarning(userId, "MEDIUM", keywords);
        }
    }
}
```

#### AI聊天集成
```java
@PostMapping("/chat")
public Result chat(...) {
    // 情绪分析
    emotionAnalysisService.analyzeAndWarn(userId, message);
    
    // 正常聊天逻辑
    String reply = aiChatService.chat(userId, sessionId, message);
    return Result.success(reply);
}
```

### 数据库表

#### user_warning_record（用户预警记录表）
```sql
CREATE TABLE user_warning_record (
    id BIGINT PRIMARY KEY AUTO_INCREMENT,
    user_id BIGINT NOT NULL,
    risk_level VARCHAR(20) NOT NULL,  -- HIGH/MEDIUM/LOW
    keywords VARCHAR(500),            -- 触发关键词
    status TINYINT DEFAULT 0,         -- 0待处理/1已处理
    handle_remark VARCHAR(500),       -- 处理备注
    handler_id BIGINT,                -- 处理人ID
    create_time DATETIME,
    update_time DATETIME
);
```

## 使用示例

### 场景1：用户在AI聊天中表达自杀倾向

**用户消息**: "我最近压力很大，感觉活着没意思，想结束这一切"

**系统处理**:
1. 检测到高风险关键词："活着没意思"、"结束"
2. 创建高风险预警（HIGH）
3. 记录关键词："活着没意义, 结束生命"
4. 管理员在预警列表中看到该用户

### 场景2：用户测试结果为"建议沟通"

**测试结果**: 分数85，结果"建议沟通"

**系统处理**:
1. 更新用户健康状态为2（建议沟通）
2. 创建预警记录
3. 推荐心理疏导科普文章和减压活动
4. 建议用户与AI咨询师聊聊

### 场景3：用户连续多次测试结果恶化

**测试历史**:
- 第1次：60分（良好）
- 第2次：75分（良好）
- 第3次：85分（建议沟通）
- 第4次：90分（建议沟通）
- 第5次：95分（建议沟通）

**系统处理**:
1. 检测到连续3次"建议沟通"
2. 创建高风险预警
3. 标记为"健康状态持续恶化"
4. 管理员需要紧急干预

## 注意事项

1. **隐私保护**: 预警信息仅限管理员查看，严格保密
2. **及时处理**: 高风险预警应在24小时内处理
3. **专业干预**: 系统只提供预警，实际干预需要专业心理咨询师
4. **误报处理**: 允许管理员标记误报，优化关键词库
5. **定期优化**: 根据实际使用情况调整关键词和判定标准
