# 登录失败问题修复指南

## 问题现象

登录时显示 "系统异常,请稍后重试"

## 问题原因

新增的业务联动功能需要数据库表结构变更,但数据库可能还没有执行变更脚本,导致:
1. User 实体类新增了 `points` 和 `recommend_preference` 字段
2. 数据库表中还没有这些字段
3. MyBatis-Plus 查询时字段不匹配导致异常

## 解决方案

### 方案一: 执行数据库修复脚本(推荐)

1. **打开MySQL命令行或Navicat**

2. **执行修复脚本**:
```bash
# 方法1: 命令行执行
mysql -u root -p youth_mental_health < fix_database.sql

# 方法2: 在Navicat中打开 fix_database.sql 并执行
```

3. **验证执行结果**:
```sql
-- 检查字段是否添加成功
DESC sys_user;

-- 应该能看到 points 和 recommend_preference 字段
```

### 方案二: 手动执行SQL

如果方案一不行,手动执行以下SQL:

```sql
USE youth_mental_health;

-- 1. 添加 points 字段
ALTER TABLE sys_user ADD COLUMN points INT DEFAULT 0 COMMENT '用户积分' AFTER recommend_preference;

-- 2. 添加 recommend_preference 字段
ALTER TABLE sys_user ADD COLUMN recommend_preference VARCHAR(50) DEFAULT NULL COMMENT '推荐偏好' AFTER health_status;

-- 3. 创建新表
CREATE TABLE IF NOT EXISTS user_learning_record (
    id BIGINT PRIMARY KEY AUTO_INCREMENT,
    user_id BIGINT NOT NULL,
    content_type TINYINT NOT NULL,
    content_id BIGINT NOT NULL,
    action_type TINYINT NOT NULL,
    duration INT DEFAULT 0,
    create_time DATETIME DEFAULT CURRENT_TIMESTAMP,
    INDEX idx_user_id (user_id)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4;

CREATE TABLE IF NOT EXISTS user_recommend_record (
    id BIGINT PRIMARY KEY AUTO_INCREMENT,
    user_id BIGINT NOT NULL,
    recommend_type TINYINT NOT NULL,
    content_id BIGINT NOT NULL,
    reason VARCHAR(200) DEFAULT NULL,
    is_clicked TINYINT DEFAULT 0,
    create_time DATETIME DEFAULT CURRENT_TIMESTAMP,
    INDEX idx_user_id (user_id)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4;

CREATE TABLE IF NOT EXISTS user_points_record (
    id BIGINT PRIMARY KEY AUTO_INCREMENT,
    user_id BIGINT NOT NULL,
    points INT NOT NULL,
    source_type TINYINT NOT NULL,
    source_id BIGINT DEFAULT NULL,
    description VARCHAR(200) DEFAULT NULL,
    create_time DATETIME DEFAULT CURRENT_TIMESTAMP,
    INDEX idx_user_id (user_id)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4;

CREATE TABLE IF NOT EXISTS user_warning_record (
    id BIGINT PRIMARY KEY AUTO_INCREMENT,
    user_id BIGINT NOT NULL,
    risk_level VARCHAR(20) NOT NULL,
    keywords VARCHAR(500) DEFAULT NULL,
    status TINYINT DEFAULT 0,
    handle_remark VARCHAR(500) DEFAULT NULL,
    handler_id BIGINT DEFAULT NULL,
    create_time DATETIME DEFAULT CURRENT_TIMESTAMP,
    update_time DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
    INDEX idx_user_id (user_id)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4;
```

## 常见问题

### Q1: 执行SQL时报错 "字段已存在"

**A**: 这是正常的,说明字段已经添加过了,可以忽略这个错误继续执行。

### Q2: 后端启动报错

**A**: 检查后端日志,可能的原因:
- 数据库连接配置错误
- 端口被占用
- Maven依赖问题

### Q3: 前端仍然登录失败

**A**: 
1. 清除浏览器缓存(Ctrl+Shift+Delete)
2. 检查后端是否正常启动
3. 查看浏览器控制台和后端日志的具体错误信息

### Q4: 如何查看后端日志?

**A**: 
- 如果在IDEA中运行,查看控制台输出
- 如果命令行运行,查看终端输出
- 查找 `ERROR` 或 `Exception` 关键字
