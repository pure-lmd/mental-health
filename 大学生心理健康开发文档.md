## 一、项目概述

### 1.1 项目简介

本系统是一个基于 **Spring Boot + Vue** 的青少年心理健康管理系统，旨在为青少年提供心理健康知识科普、心理测试评估、心理活动参与、AI心理咨询等服务。系统分为管理后台和用户前台两部分，支持管理员和普通用户两种角色。

### 1.2 核心功能

| 功能模块 | 功能描述 |
|---------|---------|
| 心理问题浏览 | 百科形式展示心理健康相关知识信息 |
| 心理问卷测试 | 用户可进行心理健康评估测试，系统自动计算得分并给出建议 |
| 系统公告 | 发布心理健康科普文章和系统通知 |
| 社区论坛 | 用户可发布动态、评论交流 |
| AI助手 | 智能AI进行心理咨询交流 |

#### 3.1.3 心理测试管理

| 功能 | 说明 |
|-----|-----|
| 测试卷管理 | 创建、编辑、删除测试卷 |
| 测试题管理 | 为测试卷添加测试题目 |
| 选项分值设置 | 为每个选项设置分值（如A:1分, B:2分, C:3分, D:5分） |
| 结果阈值配置 | 配置评估结果阈值（默认健康阈值60，良好阈值80，满分100） |

**测试评分规则**：

系统采用**归一化评分算法**，将原始分按比例映射到试卷满分（默认100分）：

```
归一化得分 = 原始得分 / 最大可能原始分 × 试卷满分
```

例如：10道题，每题最高分5分，全选最差选项时：原始分=50，归一化后 = 50/50 × 100 = **100分**

根据归一化后的得分与阈值比较，给出评估结果：
- 归一化得分 ≥ goodThreshold（80）：**建议沟通** 🔴（需要心理关注）
- 归一化得分 ≥ healthThreshold（60）：**良好** 🟡（建议适当关注情绪变化）
- 归一化得分 < healthThreshold（60）：**健康** 🟢（心理状态良好）

### 4.3 数据库E-R关系说明

| 关系 | 说明 |
|-----|-----|
| 用户 → 测试记录 | 一对多，一个用户可以有多次测试记录 |
| 测试卷 → 测试题 | 一对多，一份试卷包含多道题目 |
| 测试题 → 测试选项 | 一对多，一道题目有多个选项 |
| 用户 → 活动报名 | 一对多，一个用户可报名多个活动 |
| 活动 → 活动报名 | 一对多，一个活动可有多人报名 |

```json

  "data": {}
}
```

|-------|-----|
| 200 | 操作成功 |

| 403 | 禁止访问 |
| 404 | 资源不存在 |
| 500 | 服务器错误 |
| 1001 | 参数错误 |
| 1002 | 用户名已存在 |

| 1006 | 文件类型不支持 |

### 8.2 注意事项

1. **心理测试评分逻辑**：
   - 每道题有多个选项，每个选项设置不同分值（如A:1分, B:2分, C:3分, D:5分）
   - 用户答完所有题目后，系统计算原始总分
   - 系统自动计算每道题最高分之和（最大可能原始分），将原始总分**归一化**到试卷满分（默认100）
   - 归一化公式：`归一化得分 = 原始得分 / 最大可能原始分 × 试卷满分`
   - 根据归一化后的得分与阈值比较，给出评估结果（健康/良好/建议沟通）
   - 将评估结果同步更新到用户表的`health_status`字段

2. **AI助手对接**：
   - 基于 **Spring AI** 框架，对接**阿里云通义千问**（qwen-plus模型）
   - 使用 OpenAI 兼容模式（`https://dashscope.aliyuncs.com/compatible-mode`）
   - 支持流式响应（SSE），实现逐字输出效果
   - 每次对话通过 `session_id` 标识，支持多会话管理
   - 对话历史持久化存储到 `psy_ai_chat` 表

3. **文件上传**：
   - 通过 `FileController` 提供统一的文件上传接口
   - 上传路径：`./uploads/`，按日期自动创建子目录
   - 文件大小限制：10MB
   - 支持格式：`image/jpeg`, `image/png`, `image/gif`, `image/webp`
   - 上传后返回文件访问路径，通过静态资源映射访问

4. **安全性**：
   - 使用 **JWT** 进行身份认证，Token 通过 `Authorization` 请求头传递
   - Token 有效期：24小时（86400000ms）
   - 管理端接口通过 `AdminInterceptor` 拦截器校验管理员权限
   - 用户端接口通过 `UserInterceptor` 拦截器校验用户权限
   - 部分公开接口（登录、注册、科普列表等）排除在拦截器之外
   - 密码使用 `PasswordUtil` 加密存储
   - MyBatis Plus 自带防 SQL 注入

## 九、系统架构设计

### 9.1 整体架构

```
┌─────────────────────────────────────────────────────┐
│                    浏览器（用户端 / 管理端）              │
└──────────────────────┬──────────────────────────────┘
                       │ HTTP请求
┌──────────────────────▼──────────────────────────────┐
│              Vite Dev Server (端口: 5173)             │
│         代理 /api → localhost:8080                    │
│         代理 /uploads → localhost:8080                │
└──────────────────────┬──────────────────────────────┘
                       │
┌──────────────────────▼──────────────────────────────┐
│           Spring Boot 应用 (端口: 8080)               │
│  ┌─────────────────────────────────────────────┐    │
│  │  拦截器层 (AdminInterceptor/UserInterceptor)  │    │
│  │        ↓ JWT Token 验证                      │    │
│  │  控制器层 (Controller)                        │    │
│  │        ↓                                     │    │
│  │  服务层 (Service + ServiceImpl)               │    │
│  │        ↓                                     │    │
│  │  持久层 (Mapper + MyBatis Plus)               │    │
│  └─────────────────────────────────────────────┘    │
└──────────┬─────────────────────┬────────────────────┘
           │                     │
┌──────────▼──────────┐  ┌──────▼──────────────────┐
│  MySQL 8.0 (3306)   │  │  阿里云通义千问 AI API    │
│  youth_mental_health │  │  (qwen-plus模型)         │
└─────────────────────┘  └─────────────────────────┘
```

### 9.2 认证流程

```
1. 用户/管理员提交用户名+密码
2. 后端验证后生成 JWT Token（含 userId、role）
3. 前端将 Token 存入 LocalStorage
4. 后续请求通过 Axios 拦截器自动在请求头添加 Authorization
5. 后端拦截器解析 Token，验证角色权限
6. Token 过期（401）时前端自动跳转登录页
```

### 9.3 请求代理配置

前端开发服务器（Vite）通过代理转发请求到后端，避免跨域问题：

| 前端路径 | 代理目标 | 说明 |
|---------|---------|-----|
| `/api/*` | `http://localhost:8080` | 后端API请求 |
| `/uploads/*` | `http://localhost:8080` | 上传文件访问 |
