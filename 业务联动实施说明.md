# 业务联动增强功能实施说明

## 一、功能概述

本次实施为**大学生健康管理系统**增加了业务联动功能,实现了"测试→推荐→学习→成长轨迹"的完整业务闭环。

### 核心功能

1. **个性化推荐**: 根据用户测试结果推荐相应的科普文章和活动
2. **学习记录**: 记录用户浏览科普、参加活动等行为
3. **积分系统**: 用户完成测试、学习科普可获得积分
4. **成长轨迹**: 可视化展示用户心理健康成长历程
5. **预警机制**: 高风险用户AI咨询建议

## 四、业务联动流程

### 4.1 完整业务流程

```
1. 用户完成心理测试
   ↓
2. 系统计算测试结果(健康/良好/建议沟通)
   ↓
3. 更新用户健康状态
   ↓
4. 奖励10积分
   ↓
5. 根据测试结果生成个性化推荐
   - 建议沟通: 推荐心理疏导科普 + 减压活动 + AI咨询建议
   - 良好: 推荐心理健康科普 + 预防活动
   - 健康: 推荐心理成长科普 + 分享活动
   ↓
6. 用户浏览推荐的科普
   ↓
7. 记录学习行为 + 奖励5积分
   ↓
8. 用户报名参加推荐活动
   ↓
9. 记录学习行为 + 奖励20积分
   ↓
10. 用户查看成长轨迹
    - 测试成绩趋势图
    - 学习统计数据
    - 健康状态历史
```

### 4.2 积分奖励规则

| 行为 | 积分 | 说明 |
|-----|------|------|
| 完成心理测试 | +10分 | 每次完成测试 |
| 浏览科普文章 | +5分 | 每次浏览 |
| 参加心理活动 | +20分 | 报名成功 |
| 每日签到 | +2分 | 每天首次 |

## 五、部署步骤

### 5.1 后端部署

1. **执行数据库变更**:
```bash
mysql -u root -p youth_mental_health < business_linkage_enhancement.sql
```

2. **编译后端**:
```bash
cd youth-mental-health
mvn clean package
```

3. **启动后端**:

2. **启动开发服务器**:
```bash
npm run dev
```

3. **生产构建**:

## 十二、注意事项

1. **数据库执行**: 必须先执行SQL脚本,否则新表和字段不存在
2. **积分初始化**: SQL脚本中已包含积分初始化语句,将现有用户积分设为0

5. **缓存优化**: 成长轨迹数据可考虑使用Redis缓存
